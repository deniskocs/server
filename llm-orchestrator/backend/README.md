# `backend` — API и оркестрация

Здесь всё, что относится к **второму** контейнеру: **сервис бэкенда** (HTTP API, работа с каталогом конфигов на диске, вызовы к Docker/моделям — по спецификации UI).

- Код приложения, зависимости, `Dockerfile` бэкенда — в этой папке.
- Тома с конфигами и при необходимости `docker.sock` описываются в compose / install-скрипте и монтируются **в этот** образ, не в `web/`.

См. общую спецификацию: [README в `llm-orchestrator`](../README.md).
