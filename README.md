# Server

Репозиторий со сборкой и конфигурацией сервера (nginx, TLS, связанные сервисы). Ниже — как связать **Mac Studio** и **Ubuntu-сервер** в одну частную сеть через **Tailscale**, чтобы ходить по SSH и к внутренним портам без выставления их в интернет.

## Tailscale: что сделать на всех машинах

Идея: один аккаунт [Tailscale](https://tailscale.com/), одна виртуальная сеть (tailnet). На каждом устройстве установлен клиент; узлы получают адреса вида `100.x.x.x` и (опционально) DNS-имена внутри tailnet.

### Общее для всех

1. Зарегистрируйтесь и войдите в [консоль администратора](https://login.tailscale.com/admin/machines).
2. После подключения каждого узла убедитесь, что он в списке **Machines** и статус **Connected**.

### Mac Studio

1. Установите клиент: [Tailscale для macOS](https://tailscale.com/download/macos) (App Store или пакет с сайта). Либо Homebrew: `brew install tailscale` и запуск по [документации](https://tailscale.com/kb/1017/install/) для варианта без GUI при необходимости.
2. Откройте приложение, выполните вход в тот же аккаунт, включите подключение (статус «Connected»).
3. При необходимости включите **MagicDNS** и **Use Tailscale DNS** в настройках приложения или в админ-консоли (**DNS**), чтобы обращаться к серверу по имени вида `ubuntu` вместо IP.

### Ubuntu Server

1. Установите пакет (официальный скрипт):

   ```bash
   curl -fsSL https://tailscale.com/install.sh | sh
   ```

2. Поднимите интерфейс и привяжите узел к аккаунту:

   ```bash
   sudo tailscale up
   ```

   В терминале появится ссылка — откройте её в браузере и авторизуйте устройство (тот же аккаунт, что на Mac).

3. Полезные проверки:

   ```bash
   tailscale status
   tailscale ip -4
   ```

### Проверка связи

На Mac Studio (имя или IPv4 из `tailscale status` на Ubuntu):

```bash
ping <TAILSCALE_IP_ИЛИ_ИМЯ_УЗЛА_UBUNTU>
ssh user@<TAILSCALE_IP_ИЛИ_ИМЯ>
```

Обращайтесь к сервисам на Ubuntu по адресу **из подсети Tailscale** (`100.x.x.x`), а не по публичному IP, если хотите ограничить доступ только tailnet.

### Файрвол (Ubuntu)

Если используется `ufw`, по умолчанию трафик часто уже разрешён; если что-то режет только «внешние» интерфейсы, убедитесь, что правила не блокировали интерфейс **tailscale0**. При необходимости разрешите SSH только из tailnet, например:

```bash
sudo ufw allow in on tailscale0
```

Точные правила зависят от вашей текущей конфигурации `ufw`.

### Разное

- **Ключи API / ACL**: при команде из нескольких людей настройте [ACL](https://login.tailscale.com/admin/acls) в консоли.
- **Exit node / подсети**: нужны реже; см. [роутинг в Tailscale](https://tailscale.com/kb/1019/subnets), если надо пробросить не только сам сервер, но и локальную LAN за ним.

Подкаталоги с отдельными образами и инструкциями: `base/`, `vllm/`, `download/`.
